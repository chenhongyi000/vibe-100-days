Vibe 100天挑战 Day 1：让杂乱文件一秒拥有姓名

100 天，用 AI 持续创作。Day 1 的成果：一个能读懂照片"内心"的重命名神器。

开篇：你的文件管理，是不是也这样？

先来做个小调查——打开你的电脑，看看桌面或者照片文件夹，是不是长这样：

IMG_20231005_142356.jpg
IMG_20231005_142357.jpg
IMG_20231005_142358.jpg
新建文件夹 (3)/
最终版.docx
最终版2.0.docx
最终版_真的最终版.docx

如果你点了点头，别担心，你并不孤单。

相机导出的照片全是毫无语义的编号，甲方发来的资料包永远叫"新建文件夹"，找一张三个月前的照片得翻半天……这不是你的问题，而是传统文件管理方式的锅。

所以，Vibe 100 天挑战的第一天，我决定做一个"有灵魂"的小工具。

产品定位：不只是重命名，而是"让文件会说话"

市面上的批量重命名工具不少，比如 Total Commander、Advanced Renamer。但它们有一个共同的问题：学习成本高。正则表达式、通配符、宏变量……对普通用户来说，这些概念足够劝退。

我想要的工具是这样的：

用户不需要懂代码，只需要告诉它"我想按什么规则排列"，它就能自动预览效果，一键执行。

于是我给它取了一个名字——CleverName（灵名）。

Slogan 就叫：让杂乱无章的照片和文件，一秒拥有会说话的姓名。

核心功能：三步搞定一切

Step 1：拖入文件夹，自动识别

打开终端，把文件夹拖进去，程序自动递归扫描所有文件。

支持按类型过滤：仅图片、仅视频、图片+视频、全部文件。扫描结果会显示文件数量和类型分布。

📁 请将文件/文件夹拖入此处，或输入路径：
> C:\Photos\2024\

✅ 扫描完成：共找到 42 个文件
   📷 图片: 35  |  🎬 视频: 7

Step 2：填空式配置规则

不需要写一行代码，通过选择题就能构建命名规则：

📐 请选择命名模板：
[1] 日期_计数器     → 2024-01-15_143022_001.jpg
[2] 日期_原名       → 2024-01-15_143022_IMG_1234.jpg
[3] AI描述_日期     → 金毛犬_草地_2024-01-15_143022.jpg
[4] 自定义组合

计数器位数、日期格式都可以自定义。选"自定义组合"的话，还能自由排列 token：{date}_{original}_{counter}。

Step 3：预览确认，安心执行

程序会打印一张清晰的表格，展示"原文件名 → 新文件名"的对照清单：

                                 📋 重命名预览
┌────┬──────────────┬─────┬──────────────────────────────────┐
│  # │ 原文件名     │ →   │ 新文件名                         │
├────┼──────────────┼─────┼──────────────────────────────────┤
│  1 │ IMG_1.jpg    │ →   │ 2024-01-01_140000_001.jpg        │
│  2 │ IMG_2.jpg    │ →   │ 2024-01-01_140500_002.jpg        │
│ ...│ ...          │ ... │ ...                              │
└────┴──────────────┴─────┴──────────────────────────────────┘

确认无误后，输入 y 执行。进度条唰地跑完，搞定！

加分项：AI 语义标签 + 一键回滚

🤖 AI 看懂你的照片

如果配置了 OpenAI API Key 或者本地 Ollama，程序会自动识别图片内容，生成中文关键词。

比如一张金毛犬在草地上的照片，会自动生成标签 金毛犬_草地，作为文件名的一部分。这样不用打开图片，光看文件名就知道内容了。

没有配置 AI 也没关系，程序会自动降级为纯规则模式，不影响使用。

↩️ 一键回滚，不怕改错

万一改完不满意？程序会在文件夹里自动生成三个文件：

  • rename_mapping.json — 完整的改名映射记录
  • restore.py — Python 回滚脚本
  • restore.bat — 双击就能恢复原名！

真正做到了"改得放心，回得安心"。

技术选型：为什么选这些？

技术 用途

Day 1 的收获

这次 Vibe Coding 体验让我深刻感受到一点：

AI 不是替代你思考，而是帮你把"想法"加速变成"产品"。

从确定产品定位、设计交互流程，到写代码、跑测试，整个过程 AI 帮我省掉了大量"查文档、写样板代码"的时间。我只需要专注于"这个工具应该怎么用起来最顺手"，剩下的交给 AI 就行了。

当然，PM 思维也一直在提醒我：如果 200 张照片都是同一天同一地点拍的，按日期命名会全部撞车。 所以程序里强制要求计数器参与命名，冲突时自动追加 _v1、_v2。这个问题留到 Day 2 继续优化——比如按拍照时间间隔自动分组归档。
