Vibe 100天挑战 Day 5：我做了一个 Markdown 秒变公众号排版的小工具

Vibe Coding 100 天挑战 · Day 5

今天做的工具叫 wechat-md。

它解决的问题很简单：

我用 Markdown 写完文章之后，不想再手动给公众号重新排版。

为什么要做它？

公众号写作有一个很微妙的痛点：

Markdown 很适合写作。

但公众号编辑器并不天然适合 Markdown。

写的时候是这样的：

markdown
## 它解决什么问题？

> 文章写完之后，最烦人的不是修改内容，而是重新排版。

- 标题要调
- 引用要调
- 代码块要调
- 表格还要调

复制到编辑器之后，经常变成：

  • 标题层级丢了
  • 引用块不明显
  • 代码块不好看
  • 表格像临时拼出来的
  • 还要手动调字号、行高、边距

这不是创作，这是收尾阶段的体力活。

所以 Day 5 我决定做一个小工具：

把 Markdown 转成带内联样式的 HTML，直接复制到公众号编辑器。

这次的 MVP

我没有一上来做大而全的编辑器。

Day 5 的目标是一个可以每天用的小工具：

  • 输入一个 .md 文件
  • 输出一个 .html 文件
  • 支持公众号文章最常见的元素
  • 样式写成内联 CSS，复制时更稳定
  • 不依赖第三方库，拿来就能跑

最终命令长这样：

bash
python src/wechat_md.py examples/sample_article.md -o examples/sample_article.html

如果只想要正文片段，也可以这样：

bash
python src/wechat_md.py examples/sample_article.md --body-only -o examples/sample_body.html

它现在支持什么？

第一版支持这些能力：

元素状态
Frontmatter 标题已支持
H1-H4 标题已支持
段落已支持
引用块已支持
有序 / 无序列表已支持
行内代码已支持
围栏代码块已支持
链接和图片已支持
Markdown 表格已支持

这基本覆盖了我写公众号文章的 80% 场景。

一个小但关键的决策

很多 Markdown 转 HTML 工具会生成这样的结构：

html
<style>
h2 {
  color: #333;
}
</style>

但公众号编辑器里，复制粘贴时外部样式不一定稳定。

所以我这次选择了更朴素但更实用的方案：

html
<h2 style="margin:26px 0 14px;padding-left:12px;border-left:4px solid #07c160;">
  标题
</h2>

也就是把样式直接写到每个元素上。

这不一定优雅，但很适合公众号排版这个具体场景。

实现思路

这个工具没有引入第三方 Markdown 解析库，而是手写了一个轻量解析器。

原因也很简单：

Day 5 的目标不是兼容所有 Markdown 方言，而是解决我的高频写作场景。

核心流程分成 4 步：

  1. 读取 Markdown 文件
  2. 解析可选的 frontmatter
  3. 把常用 Markdown 块转换成 HTML
  4. 给每个 HTML 元素加上公众号友好的内联样式

例如遇到引用：

markdown
> 写完文章之后，最烦人的不是内容，而是排版。

会变成：

html
<blockquote style="...">
  写完文章之后，最烦人的不是内容，而是排版。
</blockquote>

遇到代码块时，会先转义 <、> 等特殊字符，避免代码被浏览器当成 HTML 执行。

这一天的收获

Day 5 最大的感受是：

工具不一定要复杂，关键是能不能嵌进真实流程。

这个项目不炫技。

但它非常适合我接下来的 100 天挑战。

因为之后每天都要写文章，而每篇文章都要排版。

如果一个工具能每天帮我省 10 分钟，那 100 天就是 1000 分钟。

这就是 Vibe Coding 最有意思的地方：

不是为了写代码而写代码，而是把自己每天重复做的事情，一点点变成工具。
